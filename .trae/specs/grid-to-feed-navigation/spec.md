# 网格视图点击跳转视频流 - Product Requirement Document

## Overview
- **Summary**: 在媒体库网格视图（Poster Wall）中，用户点击任意视频海报后，应用切换到视频流（Feed）模式，并从该被点击的视频开始播放，提供与 EmbyX 一致的用户体验。
- **Purpose**: 解决网格视图与视频流之间的无缝衔接问题，让用户可以从网格浏览快速定位并播放感兴趣的视频，提升浏览效率和使用体验。
- **Target Users**: 使用 EmbyTok 浏览个人媒体库的所有用户，尤其是喜欢先通过封面墙筛选再播放的用户。

## Goals
- 用户在网格视图点击任意视频后，切换到视频流模式并从该视频开始播放
- 视频流模式下的数据与网格视图数据保持一致，点击的视频在视频流中位置正确
- 切换过程流畅，无明显闪烁或卡顿
- 支持从视频流切回网格视图时保持之前的滚动位置

## Non-Goals (Out of Scope)
- 不改变网格视图的布局和样式
- 不改变视频流的播放逻辑和交互
- 不新增视频预加载策略（现有预加载逻辑保持不变）
- 不涉及多媒体库选择逻辑的修改

## Background & Context
- EmbyX（参考项目）实现了 Poster Wall 网格视图，点击视频后切换到 TikTok 风格的视频流并从该视频开始播放
- EmbyTok 已实现网格视图和视频流两种模式，通过 E 键或按钮切换
- 之前存在的问题：网格视图点击视频后，视频流总是从第一个视频开始播放，而非用户点击的视频
- 根因分析：grid → feed 切换时，`_initialItemId` 未被清理，导致初始位置跳转逻辑干扰了选中视频的跳转
- 技术栈：Flutter + Riverpod 状态管理 + PageView 视频流

## Functional Requirements
- **FR-1**: 网格视图点击视频海报触发视图切换
  - 用户在网格视图中点击任意视频海报卡片
  - 应用自动从网格模式切换到视频流模式
  - 视频流从用户点击的那个视频开始播放
- **FR-2**: 视频流与网格视图数据一致性
  - 视频流模式下的视频列表与网格视图使用同一数据源
  - 用户点击的视频在视频流中的索引位置与在网格中的全局位置一致
  - 支持分页模式下跨页跳转
- **FR-3**: 视图切换状态管理
  - grid → feed 切换时，清理初始播放位置标记，避免干扰跳转逻辑
  - grid → feed 切换时，重置滚动位置恢复标记
  - feed → grid 切换时，执行"神之一手"裁剪，显示当前播放位置所在页
- **FR-4**: 网格滚动位置保持
  - 从视频流切回网格视图时，保持网格之前的滚动位置
  - 从网格切到视频流再切回网格时，滚动位置不丢失

## Non-Functional Requirements
- **NFR-1**: 切换响应速度
  - 从点击网格视频到视频流显示目标视频的时间 < 300ms
  - 页面切换动画平滑，使用 300ms 缓动动画
- **NFR-2**: 稳定性
  - 切换过程中不出现崩溃、白屏或数据丢失
  - 在各种网络条件下（正常/慢速/断网）切换行为一致
- **NFR-3**: 可维护性
  - 跳转逻辑集中管理，易于理解和维护
  - 关键节点有日志输出，便于排查问题

## Constraints
- **Technical**: Flutter 3.x + Riverpod 2.x，必须使用现有的状态管理架构
- **Business**: 保持与 EmbyX 一致的用户体验作为参考标准
- **Dependencies**: 依赖 `videoListProvider`、`viewModeProvider`、`gridSelectedItemIdProvider` 等现有 Provider

## Assumptions
- 网格视图和视频流共享同一视频数据源（`videoListProvider` 的 `items` 列表）
- 分页模式下，`gridStartIndex` 正确反映网格视图数据在全量数据中的偏移
- 用户点击的视频 ID 一定存在于当前 `items` 列表中
- PageView 在切换视图时已正确初始化

## Acceptance Criteria

### AC-1: 网格点击视频后正确跳转播放
- **Given**: 用户已登录并进入网格视图模式，网格中显示多个视频
- **When**: 用户点击网格中第 N 个视频海报
- **Then**: 应用切换到视频流模式，并从第 N 个视频开始播放，视频画面与用户点击的海报一致
- **Verification**: `human-judgment`
- **Notes**: 可通过对比视频标题和封面确认是否为正确的视频

### AC-2: 分页模式下跨页点击跳转正确
- **Given**: 媒体库视频数 > 150，处于分页模式，用户在第 2 页
- **When**: 用户点击第 2 页中的任意视频
- **Then**: 切换到视频流后，播放的是该视频在全量列表中的对应位置，而不是第 1 页的第 N 个
- **Verification**: `human-judgment`
- **Notes**: 需要验证 `gridStartIndex` 被正确计算到跳转索引中

### AC-3: 视频流列表数据完整
- **Given**: 网格视图中有 N 个视频
- **When**: 从网格切换到视频流
- **Then**: 视频流中可以上下滑动浏览所有 N 个视频，顺序与网格视图一致
- **Verification**: `human-judgment`

### AC-4: 从视频流切回网格保持滚动位置
- **Given**: 用户在网格视图中向下滚动了一段距离，然后点击视频进入视频流
- **When**: 用户按 E 键或点击按钮切回网格视图
- **Then**: 网格视图保持之前的滚动位置，用户可以从离开的位置继续浏览
- **Verification**: `human-judgment`

### AC-5: 切换过程无闪烁卡顿
- **Given**: 应用正常运行
- **When**: 在网格和视频流之间反复切换
- **Then**: 切换过程平滑，无白屏、无明显卡顿、无视频闪烁
- **Verification**: `human-judgment`

### AC-6: 多次切换后状态一致
- **Given**: 应用正常运行
- **When**: 连续进行 10 次 grid ↔ feed 切换，每次切换前点击不同的视频
- **Then**: 每次切换后播放的视频都是用户上一次点击的视频，不会出现跳转到第一个视频的情况
- **Verification**: `human-judgment`

### AC-7: 初始跳转不影响 grid→feed 跳转
- **Given**: 用户从其他页面（如历史记录页）点击视频跳转到 FeedView，`initialItemId` 被设置
- **When**: 用户切换到网格视图，再点击另一个视频切回视频流
- **Then**: 视频流播放的是网格中点击的视频，而不是最初从其他页面跳转来的那个视频
- **Verification**: `human-judgment`
