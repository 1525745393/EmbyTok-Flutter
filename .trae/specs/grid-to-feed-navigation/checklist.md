# 网格视图点击跳转视频流 - Verification Checklist

## 基础功能验证
- [x] 网格视图中点击任意视频海报，应用切换到视频流模式
- [x] 切换后播放的视频与用户点击的视频一致（标题、封面匹配）
- [x] 视频流中可以上下滑动浏览所有视频

## 分页模式验证
- [ ] 单库模式且视频数 > 150 时，分页控件正常显示
- [x] 点击第 2 页的视频，切换到视频流后播放正确的视频（而非第 1 页的第 N 个）
- [ ] 上一页/下一页按钮功能正常

## 数据一致性验证
- [x] 视频流中的视频数量与网格视图一致
- [x] 视频流中的视频顺序与网格视图一致
- [x] 第 N 个视频在两种模式下是同一个视频

## 滚动位置保持验证
- [x] 从视频流切回网格视图时，网格滚动位置保持不变
- [x] 网格向下滚动后点击视频进入视频流，再切回网格，滚动位置正确
- [x] 反复切换视图，滚动位置不丢失

## 状态正确性验证
- [x] 从其他页面（如历史记录）跳转进入 FeedView 后，切换到 grid 再点击视频返回 feed，播放的是点击的视频
- [x] 连续切换 10 次 grid ↔ feed，每次点击不同视频，跳转位置始终正确
- [x] 不会出现跳转到第一个视频的情况

## 不同 FeedType 验证
- [x] 最新（latest）模式下点击跳转正常
- [x] 随机（random）模式下点击跳转正常
- [x] 收藏（favorites）模式下点击跳转正常
- [x] 继续观看（resume）模式下点击跳转正常
- [x] 推荐（recommend）模式下点击跳转正常

## 切换体验验证
- [x] 切换过程平滑，无白屏闪烁
- [x] 页面切换动画流畅（300ms 缓动）
- [x] 切换过程中无崩溃、无异常报错
- [ ] 慢速网络环境下切换行为正常

## 神之一手验证
- [x] feed → grid 切换时，显示当前播放视频所在的页
- [x] gridStartIndex 是 kGridPageSize (150) 的整数倍
- [x] gridItems 长度不超过 150
- [x] 搜索模式下切换视图行为正确（需要刷新时不裁剪）
