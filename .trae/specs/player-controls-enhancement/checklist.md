# 播放器控制功能增强 - 验证清单

## 基础架构与状态管理
- [ ] `isPureModeProvider` 创建后能正确初始化、更新、持久化。
- [ ] `isAutoPlayProvider`（或等效实现）存在且能跨重启保持用户选择。
- [ ] `playbackRateProvider`（或等效的播放速度状态）能正确反映当前速度 6 档。
- [ ] 新 Provider 与现有 Riverpod 架构风格一致（命名、文件位置、导入方式）。
- [ ] `app_preferences.dart`（如存在）中所有新增存储键名有统一前缀，不与现有键冲突。

## 播放速度选择
- [ ] 倍速按钮点击后弹出 6 档选择面板。
- [ ] 选择面板中：当前速度高亮显示；其他速度显示正常。
- [ ] 点击任一档位后：面板关闭，视频播放速度变更，UI 文字反映新速度。
- [ ] 速度变更触发 `controller.setPlaybackSpeed()`，不重新初始化播放器。
- [ ] 长按期间强制为 2x，松开回到用户之前选择的基准速度（无论基准是哪一档）。
- [ ] 长按倍速的 2x 临时状态与基准速度状态在内部正确区分，不会互相覆盖。
- [ ] 速度变更时 Emby 服务器收到正确的播放速度上报。
- [ ] 选择面板设计风格与整体 TikTok 暗色主题一致（半透明背景、圆角、色彩一致）。
- [ ] 面板在底部导航栏上方显示，避开安全区。

## 上一集 / 下一集 切换
- [ ] 底部控制条中播放/暂停按钮两侧出现上一集/下一集图标。
- [ ] 当前在首条时上一集按钮为禁用状态（灰色/透明），不可点击。
- [ ] 当前在末条时下一集按钮为禁用状态。
- [ ] 中间条目的两个按钮均可正常点击，触发视频切换。
- [ ] 点击下一集后：当前视频触发 `PlaybackStopped` 上报；切换到新视频；新视频触发 `PlaybackStart` 上报。
- [ ] 点击上一集后：同样正确触发停止/开始上报。
- [ ] 切换过程中有渐入动画，过渡不突兀。
- [ ] 按钮触摸目标 >= 44 x 44 dp。

## 智能进度条策略
- [ ] 播放时长 < 3 分钟的视频：底部无 `_ThinProgressBar` 显示。
- [ ] 播放时长 >= 3 分钟的视频：底部 `_ThinProgressBar` 正常显示。
- [ ] 无论视频长短，拖动进度时均正常显示 `_SeekPreviewBar` 预览条与时间戳。
- [ ] `kMinDurationForProgressBar` 常量存在且值为 3 分钟。
- [ ] 视频在 3 分钟阈值边界的行为正确（如 2:59 vs 3:00）。

## 纯净模式
- [ ] 横屏全屏模式下，右侧操作栏有"纯净模式"按钮。
- [ ] 点击纯净模式按钮后：右侧操作栏、底部控制条、标题/简介等全部以 300ms 渐隐动画消失。
- [ ] 纯净模式下：单击屏幕，所有 UI 重新渐显。
- [ ] 纯净模式状态持久化：重启应用后保留上次选择。
- [ ] 竖屏流式浏览中纯净模式默认关闭（不会意外隐藏右侧点赞按钮）。
- [ ] 动画流畅，无闪烁/撕裂。

## 自动播放
- [ ] 底部控制条中有自动播放开关（图标 + 文字反映 On/Off 状态）。
- [ ] 点击切换：立即改变状态并持久化。
- [ ] 自动播放开启时：当前视频播放完成后，自动播放下一条。
- [ ] 自动播放关闭时：当前视频播放完成后停止，不自动切换。
- [ ] 自动播放在用户主动暂停视频后不触发（即使自动播放开启）。
- [ ] 视频切换渐入动画平滑。

## TV 模式与遥控器
- [ ] 大屏设备（短边 > 800dp）自动检测为 TV 模式，提供更粗的焦点元素。
- [ ] 遥控器方向键上/下：切换视频上一条/下一条。
- [ ] 遥控器方向键左/右：快退/快进 10 秒，进度条反映新位置。
- [ ] 遥控器 OK/Enter：播放/暂停切换。
- [ ] 遥控器 ESC/返回：退出全屏或返回上级视图。
- [ ] 当前聚焦元素有明显视觉反馈（边框高亮 + 轻微缩放）。
- [ ] TV 模式下字体/间距比标准模式大 20% 左右。
- [ ] 非 TV 模式下，TV 逻辑不干扰正常触摸操作。

## 随机浏览模式
- [ ] 顶部工具栏有"随机"模式入口，与"最新""收藏"并列。
- [ ] 进入随机模式后：播放列表刷新为从当前库随机拉取的 80 条视频。
- [ ] 随机列表与"最新"模式的列表不同（除非极小库）。
- [ ] 切换动画与进入其他模式一致，体验流畅。
- [ ] `kRandomListSize` 常量存在且值为 80。

## 播放状态上报与续播
- [ ] 倍速变更时 Emby 服务器收到 `PlaybackProgress` 且反映新速度。
- [ ] 切换视频时旧视频触发 `PlaybackStopped`，新视频触发 `PlaybackStart`。
- [ ] 拖动进度（seek）后，服务器状态反映新的播放位置。
- [ ] `playbackLevelProvider` 在上述操作中保持正确（Direct Play/Transcode 状态不被误判）。
- [ ] 用户下次进入"继续观看"能看到之前离开时的续播点。

## 性能与稳定性
- [ ] 连续播放 30 分钟并频繁切换视频，应用无崩溃。
- [ ] 长时间使用后内存占用稳定（无持续增长的泄漏迹象）。
- [ ] 新增 Provider 未引起不必要的 rebuild（可通过 `debugPrint` 或 DevTools 验证）。
- [ ] `flutter analyze` 无新增警告/错误。
- [ ] 所有动画帧率保持 60fps。

## 可访问性
- [ ] 所有控制按钮的触摸目标 >= 44 x 44 dp。
- [ ] 文本与背景的对比度 >= 4.5:1（尤其 TV 模式的大字）。
- [ ] 按钮有语义标签（Semantics widget 或 tooltip），便于屏幕阅读器。

## 向后兼容
- [ ] 升级用户的原有观看历史不受影响。
- [ ] 不使用新功能时，应用表现与之前一致（各开关默认关闭或保持原行为）。
- [ ] Emby API 调用无破坏性变更（请求结构、路径、参数均兼容）。
- [ ] 降级到不支持的 Emby 服务器版本时，不会崩溃，仅可能部分上报功能不生效。
