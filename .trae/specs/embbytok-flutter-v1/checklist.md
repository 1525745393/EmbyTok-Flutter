# EmbyTok-Flutter v1 - 验证检查清单

> 本清单对应 spec.md 中 18 个 Acceptance Criteria，用于逐项验证项目是否达到发布标准。
> 状态标记：`[ ]` 未通过 · `[/]` 进行中 · `[x]` 已通过

---

## 功能验证（Functional）

### AC-1: 登录与服务器配置
- [ ] 登录页包含：后端代理地址输入框、Emby 服务器地址输入框、用户名输入框、密码输入框、可选 API Key 输入框、登录按钮
- [ ] 输入有效信息后提交，成功调用 `/api/auth/login`，收到 token 并持久化到本地存储
- [ ] 登录成功后自动跳转到视频流页面或媒体库选择页面
- [ ] 服务器地址格式错误、用户名密码错误、后端代理不可用时，展示对应的中文错误提示
- [ ] 重启应用后，已登录状态自动恢复（无需重新登录）
- [ ] 退出登录可清除本地 token 并返回登录页

### AC-2: 媒体库列表展示
- [ ] 已登录用户进入应用后可看到媒体库列表（如 "电影"、"剧集"、"音乐"）
- [ ] 每个媒体库显示：名称、封面图、项目数量
- [ ] 点击某个媒体库可切换到该库的视频流浏览
- [ ] 媒体库切换后列表内容正确刷新

### AC-3: 竖屏视频流
- [ ] 视频流页面以全屏竖向方式展示，每个视频占满一屏
- [ ] 使用 PageView（或等效组件）支持上下滑动切换视频
- [ ] 进入页面后当前视频自动开始播放（无需用户点击）
- [ ] 向上滑动切换到下一个视频：旧视频暂停 → 新视频播放
- [ ] 向下滑动回到上一个视频
- [ ] 列表触底时自动加载下一页

### AC-4: 视频播放控制
- [ ] 控制条可见：播放/暂停按钮、进度条、当前时间 / 总时长、倍速选择、全屏按钮
- [ ] 播放 / 暂停按钮状态与视频实际播放状态同步
- [ ] 拖动进度条可跳转到指定位置，松手后视频从该位置播放
- [ ] 倍速选择至少支持 0.5x / 1x / 1.5x / 2x 四档
- [ ] 点击画面切换控制条显隐；控制条 3 秒无操作后自动隐藏

### AC-5: 手势交互
- [ ] 单击画面切换播放 / 暂停
- [ ] 双击画面触发点赞动画，并将该视频加入收藏
- [ ] 长按画面期间视频以 2x 速度播放，松开恢复原速
- [ ] 水平拖动画面可快进 / 快退（拖动距离与时长成比例）
- [ ] 上述手势不与 PageView 的竖向滑动冲突

### AC-6: 智能预加载
- [ ] 当前播放第 N 个视频时，第 N+1 个视频的缩略图已在后台加载
- [ ] 当前播放第 N 个视频时，第 N+1 个视频的 MediaController 已初始化并开始缓冲
- [ ] 在稳定网络下滑动切到下一个视频：播放启动时间 < 500ms
- [ ] 网络为移动网络时预加载数量限制为 1，WiFi 时可扩展为 2

### AC-7: 收藏功能
- [ ] 双击视频或点击收藏按钮可切换收藏状态
- [ ] 收藏状态在 "我的收藏" 页面实时同步
- [ ] "我的收藏" 页面以列表形式展示所有已收藏视频（缩略图、标题、时长）
- [ ] 收藏数据通过 Emby 播放列表 API 持久化到服务器（或本地 fallback）
- [ ] 取消收藏后视频从收藏列表移除
- [ ] 空收藏状态有友好提示

### AC-8: 搜索功能
- [ ] 搜索页面包含搜索栏与结果区
- [ ] 输入关键词并提交后，调用 `/api/search` 并展示结果
- [ ] 结果按类型（电影 / 剧集 / 音乐）分组展示
- [ ] 结果数量多时支持 "加载更多" 分页
- [ ] 点击搜索结果项可进入视频播放
- [ ] 最近 10 条搜索关键词本地持久化，点击可快速重复搜索

### AC-9: 字幕支持
- [ ] 播放视频时可通过字幕按钮查看可用语言轨道列表
- [ ] 选择某条轨道后视频画面底部显示对应字幕
- [ ] 字幕内容与视频时间同步，切换时间轴后字幕内容正确变化
- [ ] 可调整字幕字体大小（至少三档）
- [ ] 可调整字幕颜色（至少白 / 黄两种）
- [ ] 可调整字幕垂直位置（底部 / 偏下 / 居中）
- [ ] 字幕样式偏好持久化

### AC-10: 播放历史与进度记忆
- [ ] 播放视频时每 30 秒（或退出时）自动保存观看进度
- [ ] 再次打开同一视频从上次进度位置继续播放
- [ ] "观看历史" 页面以时间倒序列出已观看视频
- [ ] 历史项显示缩略图、标题、观看百分比
- [ ] 点击历史项进入播放并跳转到上次位置
- [ ] 支持一键清空观看历史

## 后端 / API 验证

### AC-11: 健康检查
- [ ] `GET /health` 返回 HTTP 200，响应体为 JSON，含 `status: "ok"` 与 `version` 字段
- [ ] Swagger UI (`/docs`) 可在浏览器访问
- [ ] OpenAPI JSON (`/openapi.json`) 可访问且 schema 完整

### AC-12: 代理认证
- [ ] `POST /api/auth/login` 接收 `{emby_url, username, password}` JSON body
- [ ] 成功认证：返回 HTTP 200，JSON 含 `access_token`, `user_id`, `username`
- [ ] 密码错误或用户不存在：返回 HTTP 401 + 清晰错误信息
- [ ] Emby Server 不可达：返回 HTTP 502 或 503
- [ ] 请求体字段缺失：返回 HTTP 422（由 Pydantic 自动验证）

### AC-13: 视频列表接口
- [ ] `GET /api/libraries` 返回媒体库列表，每项含 `id`、`name`、可选 `image_url`、`item_count`
- [ ] `GET /api/libraries/{library_id}/items?limit=20&offset=0` 返回分页视频列表
- [ ] 列表项包含：`id`, `title`, `type`, `duration_seconds`, `thumbnail_url`
- [ ] `limit` / `offset` 参数生效，分页结果正确
- [ ] 请求未携带 Token 时返回 HTTP 401

### AC-14: 搜索与播放地址
- [ ] `POST /api/search` 接收 `{query, types?, limit, offset}` 返回分组搜索结果
- [ ] 空关键词返回合理错误或空列表（不崩溃）
- [ ] `GET /api/items/{item_id}` 返回视频详情
- [ ] `GET /api/items/{item_id}/playback` 返回可播放的 URL（直链或 HLS m3u8）
- [ ] 该 URL 可直接被 media_kit / video_player 成功播放

### AC-15: Docker 容器化
- [ ] `backend/Dockerfile` 存在且基于 Python 3.11 镜像
- [ ] `docker build -t embbytok-backend backend/` 构建成功
- [ ] `docker run -p 8000:8000 embbytok-backend` 启动后 `/health` 可访问
- [ ] `docker-compose up -d` 一键启动成功
- [ ] 容器日志中无致命错误

## 项目基础验证

### AC-16: Flutter 项目可运行
- [ ] `flutter pub get` 无依赖冲突或解析失败
- [ ] `flutter analyze` 无严重错误（warning 可存在）
- [ ] `flutter run` 可成功编译并安装到已连接设备 / 模拟器
- [ ] 应用启动后展示登录页（未登录态）或视频流页面（已登录态）

### AC-17: 错误处理与用户提示
- [ ] 网络请求失败时显示中文错误提示 SnackBar，不崩溃
- [ ] Emby Server 不可达时显示清晰提示（"无法连接到媒体服务器"）
- [ ] 无网络连接时顶部显示红色提示条
- [ ] 所有未捕获 Flutter error 通过 `FlutterError.onError` 统一处理
- [ ] 异步异常通过 `runZonedGuarded` 捕获
- [ ] 错误场景下有明确的重试按钮/机制

### AC-18: 代码结构与文档
- [ ] `frontend/lib/` 下包含 `models/`, `views/`, `widgets/`, `services/`, `providers/`, `utils/` 目录
- [ ] `backend/` 下包含 `main.py`, `routers/`, `models/`, `clients/`, `core/` 目录
- [ ] 每个文件职责单一，避免千行以上大文件
- [ ] 根目录 `README.md` 完整介绍：项目简介、技术栈、环境要求、安装运行、功能列表、目录结构
- [ ] README 中的所有命令经过实际执行验证

## 性能验证（非功能性）

- [ ] Flutter 客户端视频流滑动帧率在现代中端手机上稳定 > 50fps（通过 Flutter Performance 工具验证）
- [ ] 首次进入视频流页面到首个视频开始播放的时间 < 3 秒（10Mbps 网络下）
- [ ] 连续滑动 50 个视频后应用无内存泄漏（内存曲线平稳）
- [ ] FastAPI 单节点可处理至少 50 并发请求，响应延迟 < 500ms（转发 Emby 响应）

## 安全验证

- [ ] 密码明文不写入客户端本地存储（仅 token 持久化）
- [ ] Token 通过 HTTP Authorization header 或自定义 header 传递，不在 URL query 中
- [ ] FastAPI 日志中不输出 password/token 等敏感字段
- [ ] HTTPS 部署下 CORS 配置严格（不使用 `*`）
