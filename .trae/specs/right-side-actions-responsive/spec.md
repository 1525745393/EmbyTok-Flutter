# 右侧操作区响应式优化 - Product Requirement Document

## Overview
- **Summary**: 将右侧操作区从当前的固定像素布局（96px 容器 + 48px 按钮）重构为基于屏幕宽度的响应式布局，确保在手机、平板和桌面浏览器上都有合适的按钮尺寸和点击区域。
- **Purpose**: 当前右侧操作区仅针对手机竖屏（~360-480px 宽）设计，在桌面端浏览器（> 800px 宽）上按钮尺寸过小，点击体验不佳；同时在平板和大屏手机横屏模式下布局比例失衡。本优化旨在根据屏幕尺寸动态调整按钮大小、容器宽度和间距，使所有设备上都有良好的用户体验。
- **Target Users**: 所有使用 EmbyTok 的用户，尤其关注大屏幕（桌面浏览器、平板横屏、电视模式）和小屏幕手机的使用体验。

## Goals
1. **按钮尺寸响应式**: 根据屏幕宽度自动调整按钮尺寸（圆形按钮直径、图标大小、标签字号），保证点击目标大小符合设计规范（建议至少 48dp 点击目标）
2. **容器宽度响应式**: 右侧操作区容器宽度根据屏幕宽度比例调整，避免在大屏上显得过于局促或过小
3. **间距响应式**: 按钮间垂直间距、按钮与容器边缘的内边距根据屏幕宽度自适应
4. **保持移动端视觉一致**: 手机竖屏下视觉效果与当前保持一致（不改变现有用户习惯）
5. **桌面端明显增强**: 宽屏（> 800px）上按钮尺寸和容器宽度适当放大（+ 30-50%），提升点击便利性和视觉平衡感
6. **代码可维护性**: 将所有硬编码像素值抽取为响应式计算或常量，便于后续调整和维护

## Non-Goals (Out of Scope)
- **不改变按钮排列顺序**: 保持当前海报头像 → 点赞 → 信息 → 删除 → 倍速 → 播放模式 → 字幕 → 唱片 → 全屏的顺序
- **不改变按钮功能和交互方式**: 不改变点击行为、动画效果、快捷键和焦点管理
- **不添加新按钮或移除现有按钮**: 按钮数量和功能保持不变
- **不改变色彩方案和视觉风格**: 颜色配置、图标选择保持与当前一致
- **不实现列溢出滚动**: 预计响应式放大后按钮列不会超出屏幕高度，暂不处理滚动
- **不改变底部信息条和其他区域**: 仅优化右侧操作区，不影响其他 UI 组件

### 设备适配问题分析

#### 手机竖屏（宽度 320-480px）
- 当前状态：正常 ✓
- 容器 96px 占屏幕宽度的 20-27%，合理
- 按钮 48px 为舒适的点击区域

#### 平板 / 大屏手机横屏（宽度 600-900px）
- 问题：容器 96px 仅占屏幕宽度的 10-15%，**视觉上过小**
- 问题：按钮 48px 在平板上点击目标相对较小
- 建议：按钮和容器放大 1.3-1.5 倍

#### 桌面浏览器（宽度 900-1920+px）
- 问题：容器 96px 仅占屏幕宽度的 **5-10%**，**视觉上严重失衡**
- 问题：按钮 48px 与宽大的视频画面对比不协调
- 问题：点击目标过小，用户操作困难
- 建议：按钮和容器放大 1.5-1.8 倍

### Flutter 响应式布局最佳实践
- `MediaQuery.of(context).size` 获取屏幕尺寸
- `MediaQuery.of(context).size.shortestSide` 或 `.width` 用于断点判断
- `LayoutBuilder` 获取父组件的可用空间
- 可使用 `Theme.of(context).textScaleFactor` 考虑用户文字缩放设置
- 断点建议参考 Material Design 响应式指南：手机 < 600, 平板 600-840, 桌面 > 840

## Functional Requirements

### FR-1: 容器宽度响应式
右侧操作区容器宽度不再固定为 96px，而是根据屏幕宽度计算：
- 屏幕宽度 ≤ 480px（手机）：容器宽度 = **96px**（保持当前值）
- 屏幕宽度 481-800px（大屏手机/小平板）：容器宽度 = **112-128px**
- 屏幕宽度 801-1200px（平板/桌面）：容器宽度 = **128-160px**
- 屏幕宽度 > 1200px（大桌面）：容器宽度 = **160-192px**
- 计算公式建议：`min(max(screenWidth * 0.18, 96), 192)`

### FR-2: 圆形按钮尺寸响应式
所有 48×48 的圆形按钮（海报、删除、倍速、播放模式、字幕、唱片静音）需要响应式调整：
- 屏幕宽度 ≤ 480px：按钮直径 **48px**（保持当前值）
- 屏幕宽度 481-800px：按钮直径 **56-64px**
- 屏幕宽度 801-1200px：按钮直径 **64-72px**
- 屏幕宽度 > 1200px：按钮直径 **72-80px**
- 内部图标大小按比例调整（当前 22-28px → 最大 36-40px）

### FR-3: 通用按钮（_PressableActionButton）尺寸响应式
- 图标：32 → 最大 40-44px
- 标签文字：12pt → 最大 14-16pt
- 标签与图标间距：4 → 最大 6px
- 按钮内边距同步放大

### FR-4: 间距响应式
- 按钮间垂直间距：20px → 根据屏幕宽度最大 28-32px
- 容器左右内边距：8 → 最大 12-16px
- 容器顶部内边距（偏移量）：保持现有逻辑但随容器宽度放大微调
- 容器底部内边距：保持现有逻辑但随容器宽度放大微调

### FR-5: 保持视觉平衡
- 按钮放大后，内部的图标大小和文字大小需要等比放大，保持视觉平衡
- 标签文字不应过大，保持清晰可读但不喧宾夺主
- 圆形按钮边框宽度：2px → 最大 3px（保持视觉比例）
- 海报头像内嵌 + 号按钮尺寸：22 → 最大 26-28px

### FR-6: 纯净模式按钮区适配
`_buildCleanModeRightActions` 中使用的 `buttonWidth: 96.0` 需要根据相同的响应式逻辑调整，与主操作区保持一致。

### FR-7: 中央播放按钮尺寸响应式（可选增强）
中央播放/暂停按钮容器（72×72, line 622-623）也可以适当响应式调整（作为额外增强项）。

## Non-Functional Requirements

### NFR-1: 性能
响应式计算应在 build 方法内进行简单的算术运算，不引入额外的异步操作或重计算。使用 Flutter 框架的 LayoutBuilder/MediaQuery 已提供的机制，不增加显著性能开销。

### NFR-2: 代码可维护性
所有响应式计算逻辑应集中管理：
- 在 `_VideoPageItemState` 中添加一个响应式工具方法（如 `_responsiveSize(double baseSize, double maxScale)`）
- 或定义一个内部的类来管理响应式断点和计算
- 避免散落在各处的 `if/else` 判断

### NFR-3: 向后兼容性
- 手机竖屏（360-480px 宽）视觉效果与修改前保持一致或几乎一致
- 不改变按钮的交互逻辑和功能
- 不改变用户已形成的操作习惯

### NFR-4: 布局稳定性
- 响应式调整后，按钮列不应出现意外的溢出（overflow）
- 按钮列在所有尺寸下都应完整显示（已有 `MainAxisAlignment.end`）
- 避免与底部信息条或中央播放按钮发生重叠

### NFR-5: TV 模式考虑
- TV 模式下焦点管理（`_isFocused`）的焦点高亮边框大小也应跟随响应式调整
- 焦点高亮边框宽度：2px → 最大 3px

## Constraints
- **技术约束**: Flutter 3.10+/Dart 3.0+，使用 `MediaQuery` 和 `LayoutBuilder`，不引入新依赖
- **设计约束**: 色彩和视觉风格保持与当前一致；仅调整尺寸相关参数
- **时间约束**: 这是一个纯 UI 调整，不涉及业务逻辑变更
- **功能约束**: 不改变按钮功能和交互方式，只改变大小

## Assumptions
- EmbyTok 在竖屏滑动视频的容器中使用，容器宽度 = 屏幕宽度（或受限宽度）
- 用户主要在以下设备尺寸区间使用：
  - 小屏手机：320-400px 宽
  - 标准手机：400-480px 宽
  - 大屏手机/小平板：481-800px 宽
  - 平板/桌面：801-1920px 宽
- 所有设备的屏幕高度都足够显示按钮列（9 个按钮 × 约 70px ≈ 630px，加上顶部和底部内边距）
- 用户的文字缩放设置（`textScaleFactor`）在正常范围内（0.85-1.3），极端值可能需要额外处理

## Acceptance Criteria

### AC-1: 容器宽度响应式调整
- **Given**: 应用在不同屏幕宽度的设备上运行
- **When**: 显示视频页面的右侧操作区
- **Then**: 容器宽度根据屏幕宽度按比例调整，手机端保持约 96px，桌面端放大至约 160-192px
- **Verification**: `human-judgment`
- **Notes**: 需在至少 3 种不同尺寸（360px、700px、1200px）上验证

### AC-2: 圆形按钮尺寸响应式调整
- **Given**: 应用在不同屏幕宽度的设备上运行
- **When**: 显示右侧操作区的圆形按钮（海报、删除、倍速、播放模式、字幕、唱片静音）
- **Then**: 按钮的直径和内部图标大小按比例放大，手机端保持约 48px，桌面端放大至约 72-80px
- **Verification**: `human-judgment`

### AC-3: 通用按钮尺寸响应式调整
- **Given**: 应用在不同屏幕宽度的设备上运行
- **When**: 显示右侧操作区的通用按钮（点赞、信息、全屏）
- **Then**: 图标大小和标签文字大小按比例放大，手机端保持与当前一致
- **Verification**: `human-judgment`

### AC-4: 按钮间距响应式调整
- **Given**: 应用在不同屏幕宽度的设备上运行
- **When**: 显示右侧操作区完整的按钮列
- **Then**: 按钮间垂直间距从手机端的 20px 放大到桌面端的约 28-32px，保持视觉上的舒适间距
- **Verification**: `human-judgment`

### AC-5: 手机端视觉一致性
- **Given**: 标准尺寸手机（360-480px 宽）
- **When**: 显示视频页面
- **Then**: 右侧操作区的视觉效果与修改前保持一致或几乎一致（用户无法轻易察觉差别）
- **Verification**: `human-judgment`

### AC-6: 桌面端视觉增强
- **Given**: 桌面浏览器（1200px+ 宽）
- **When**: 显示视频页面
- **Then**: 右侧操作区按钮明显大于手机端，点击目标更大更方便，视觉上与视频画面比例协调
- **Verification**: `human-judgment`

### AC-7: 无布局溢出或重叠
- **Given**: 任何尺寸的设备
- **When**: 显示完整的视频页面（包含右侧操作区、底部信息条、中央播放按钮）
- **Then**: 不出现布局溢出错误，按钮不与其他 UI 元素重叠，所有内容完整可见
- **Verification**: `programmatic`
- **Notes**: 使用 Flutter DevTools 的 widget inspector + 手动测试，确保在 320×480、480×800、1024×768、1920×1080 等不同尺寸下无问题

### AC-8: TV 焦点模式正常工作
- **Given**: 使用 TV 遥控器在视频页面导航
- **When**: 焦点移动到右侧操作区的按钮上
- **Then**: 焦点高亮边框正确显示在按钮周围，边框宽度也随按钮尺寸响应式调整
- **Verification**: `human-judgment`
- **Notes**: 可使用键盘方向键模拟 TV 遥控器焦点

### AC-9: 纯净模式按钮区正常显示
- **Given**: 开启自动连播（isAutoPlay=true，纯净模式）
- **When**: 显示视频页面
- **Then**: 右侧可拖拽的纯净模式按钮区（连播 + 倍速）尺寸响应式，与主操作区的缩放比例一致
- **Verification**: `human-judgment`

### AC-10: 代码可维护性
- **Given**: 代码审查者查看修改后的 `video_page_item.dart`
- **When**: 审查响应式相关代码
- **Then**: 响应式计算逻辑集中管理、命名清晰、易于理解和后续调整，而非散落在 widget 树中
- **Verification**: `human-judgment`
