# EmbyTok 响应式设计 - 产品需求文档

## Overview
- **Summary**: 在手机竖屏模式下，视频内容全屏展示，顶部工具栏和底部导航栏采用半透明渐变叠加在视频画面上，形成"内容优先、UI 轻量"的沉浸式体验。
- **Purpose**: 当前工具栏为不透明黑色背景，视频内容被工具栏遮挡，无法真正延伸到屏幕边缘。抖音/TikTok 等主流短视频应用都采用半透明叠加的设计，让视频画面更具冲击力。
- **Target Users**: 所有在手机竖屏模式下使用 EmbyTok 的用户（主要使用场景）。

## Goals
- 视频内容真正延伸到屏幕顶部和底部边缘，消除黑色条块感
- 顶部工具栏和底部导航栏使用半透明渐变背景，视觉上叠加在视频上
- 工具栏的图标和文字保持足够的对比度，确保可读性
- 保留现有的手势交互：滑动切换视频、点击唤醒工具栏、自动隐藏逻辑
- 在手机竖屏下达到最佳的视觉平衡：内容占比 ≥ 90%，UI 占比 ≤ 10%

## Non-Goals (Out of Scope)
- 不改变横屏/平板的布局（本 spec 专注手机竖屏）
- 不添加新的视频播放控制功能（如倍速、亮度调节等）
- 不修改媒体库选择器的内容，只调整外观
- 不添加新的手势或触摸效果

## Background & Context
- **当前布局**: `FeedView` 使用 `Stack` 结构：视频 PageView 在底层，`TopToolBar` 用 `AnimatedContainer` 包装在顶层。工具栏背景为不透明 `backgroundColor` (0xFF000000)，与视频内容有明显分界线。
- **技术栈**: Flutter 3.x + Riverpod + video_player，支持 Android 6.0+
- **相关常量**: `kToolbarHeight = 56`, `kBottomNavHeight = 56`, `kToolbarAnimMs = 200`
- **状态管理**: `toolbarVisibilityProvider` (StateNotifier) 控制工具栏和底部导航的显隐

## Functional Requirements
- **FR-1**: 顶部工具栏使用半透明渐变背景（自顶向下：`Color(0xAA000000)` → `Color(0x00000000)`），去除底部边框分隔线
- **FR-2**: 底部导航栏使用半透明渐变背景（自底向上：`Color(0xAA000000)` → `Color(0x00000000)`），去除顶部分隔线
- **FR-3**: 视频内容区全屏展示，`Positioned.fill` 填充整个屏幕，不被工具栏"推下去"
- **FR-4**: 顶部工具栏和底部导航栏使用 `Positioned` 绝对定位叠加在视频上方，而不是 `Column` 垂直布局
- **FR-5**: `SafeArea` 只保护工具栏和导航栏的内容区域（避开刘海、动态岛、底部手势条），不保护视频内容
- **FR-6**: 工具栏可见时，视频内容区的关键信息（右侧操作按钮、底部标题）应避开工具栏区域，确保视觉完整性
- **FR-7**: 保持现有的滑动手势和工具栏动画联动（向上滑 → 隐藏；下滑或点击 → 显示；3 秒后自动隐藏）

## Non-Functional Requirements
- **NFR-1**: 性能：半透明叠加不应导致帧率下降超过 5fps（在 Pixel 5 等中端手机上应保持 55-60fps）
- **NFR-2**: 可访问性：工具栏图标和文字的背景对比度 ≥ 4.5:1（文本）/ 3:1（图标）
- **NFR-3**: 视觉一致性：顶部和底部的渐变效果参数一致（透明度、方向）
- **NFR-4**: 代码可维护性：颜色和透明度参数应集中在常量文件，避免硬编码

## Constraints
- **技术**: Flutter 框架，不能依赖第三方模糊效果库（如 `glassmorphism`），必须使用 Flutter 原生 `LinearGradient`
- **平台**: 适配 Android 6.0+，包括刘海屏、动态岛、底部手势条的各种设备
- **向后兼容**: 不能破坏横屏/网格视图模式的现有布局
- **依赖**: 现有 Riverpod 状态管理、`SystemChrome` 系统 UI 控制

## Assumptions
- 用户在手机竖屏模式下主要关注视频内容本身，工具栏作为辅助信息，应该"退居二线"
- 半透明渐变 + 深色视频内容背景足以保持文本对比度，无需额外的文字阴影
- Android 的 `SystemUiMode.edgeToEdge` 能让内容正确延伸到状态栏下方
- 现有的手势监听器（`PageView` 的物理滚动）不会被半透明层干扰

## Acceptance Criteria

### AC-1: 顶部工具栏半透明叠加
- **Given**: 用户在手机竖屏模式下浏览视频流视图
- **When**: 工具栏可见状态为 `true`（下滑或点击画面后）
- **Then**: 顶部工具栏呈现半透明黑色渐变，视频内容透过工具栏可见，且工具栏的图标和文字清晰可读
- **Verification**: `human-judgment`
- **Notes**: 检查工具栏从顶到下的渐变是否平滑，底部边缘不应有硬边框

### AC-2: 底部导航栏半透明叠加
- **Given**: 用户在手机竖屏模式下浏览视频流视图
- **When**: 工具栏可见状态为 `true`
- **Then**: 底部导航栏呈现半透明黑色渐变，视频内容透过导航栏可见，导航项图标清晰可读
- **Verification**: `human-judgment`
- **Notes**: 底部导航栏应与顶部工具栏使用相同的透明度参数

### AC-3: 视频内容全屏延伸
- **Given**: 用户在手机竖屏模式下浏览视频流视图
- **When**: 查看任意视频页
- **Then**: 视频画面从屏幕顶部边缘延伸到底部边缘，状态栏和底部手势条覆盖在视频上，无黑色空条
- **Verification**: `human-judgment`
- **Notes**: 在带有刘海/动态岛的设备上，状态栏图标应在视频上浮动

### AC-4: 操作按钮避让工具栏
- **Given**: 工具栏可见状态为 `true`（不透明层覆盖在视频上）
- **When**: 查看视频页的右侧操作按钮（静音/点赞/收藏/评论/分享）和底部标题
- **Then**: 操作按钮避开顶部工具栏区域向下偏移 56px + 顶部安全 padding；底部标题避开底部导航栏向上偏移 56px + 底部安全 padding
- **Verification**: `human-judgment`
- **Notes**: 关键检查点：顶部的"静音"按钮不应被工具栏遮挡，底部的标题不应被底部导航栏遮挡

### AC-5: 工具栏隐藏时布局自适应
- **Given**: 工具栏可见状态为 `false`（向上滑动后或 3 秒无交互）
- **When**: 工具栏和底部导航栏折叠（高度变为 0）
- **Then**: 操作按钮和标题的 padding 自适应，顶部恢复为仅安全 padding，底部恢复为仅安全 padding；视频画面在整屏展示，无残留占位
- **Verification**: `human-judgment`
- **Notes**: toolbar 折叠后操作按钮应立即移动到合适位置，动画与 toolbar 动画联动

### AC-6: 不影响网格视图模式
- **Given**: 用户在网格视图（`ViewMode.grid`）下浏览
- **When**: 切换到网格视图
- **Then**: 网格视图保持原有布局，视频卡片保持原样，不受半透明叠加影响
- **Verification**: `human-judgment`

### AC-7: 手势交互保持不变
- **Given**: 用户在视频流视图下使用
- **When**: 执行向上滑动切换视频、向下滑动、点击画面空白区域等操作
- **Then**: 工具栏显隐逻辑与之前一致（上滑隐藏、下滑显示、点击显示+3秒自动隐藏），手势无延迟或失灵
- **Verification**: `human-judgment`

### AC-8: 不同设备适配（安全区域）
- **Given**: 在带有刘海/动态岛/底部手势条的手机上使用
- **When**: 竖屏浏览
- **Then**: 工具栏内容在 `SafeArea` 内避开刘海/动态岛，底部导航栏内容避开底部手势条，视频内容延伸到边缘但被系统 UI 部分覆盖
- **Verification**: `human-judgment`

## Open Questions
- [ ] 是否需要在半透明叠加层上加入轻微的模糊效果（类似 iOS 的 `UIVisualEffectView`）？当前方案只使用渐变，效果更轻但也更"平"
- [ ] 网格视图（`ViewMode.grid`）的顶部工具栏是否也应改为半透明，以保持一致性？当前设计仅针对 feed 视图
- [ ] 当工具栏展开时，右侧操作按钮是否需要加一个额外的文字阴影来增强在亮背景视频上的可读性？
