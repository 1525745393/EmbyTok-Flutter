# 媒体库选择器多选模式 - Product Requirement Document

## Overview
- **Summary**: 将媒体库选择器从单选模式改为多选模式，允许用户勾选多个媒体库后一次性确认，提升用户体验。
- **Purpose**: 当前单选模式用户体验不佳，用户每次只能选择一个媒体库，想看多个库的视频需要反复切换。多选模式允许用户一次选择多个媒体库。
- **Target Users**: 使用 EmbyTok 浏览个人媒体库的所有用户。

## Goals
- 用户可以勾选多个媒体库（支持全选/取消全选）
- 点击媒体库卡片仅切换选中状态，不关闭弹窗
- 显示"确认"和"取消"按钮，确认后应用选择并关闭弹窗
- 收藏夹作为特殊入口保持单选逻辑（点击即切换并关闭）
- 多选时切换到 latest 模式（因为 favorites 是单一收藏夹列表，不支持多库混合）

## Non-Goals (Out of Scope)
- 不改变媒体库卡片的视觉样式（保持现有设计）
- 不改变媒体库列表的数据加载逻辑
- 不支持在多选模式下同时浏览多个收藏夹

## Functional Requirements
- **FR-1**: 多选交互
  - 点击媒体库卡片：切换该库的选中/未选中状态，不关闭弹窗
  - 选中状态用边框颜色和背景色区分
  - 已选中的库数量显示在标题栏
- **FR-2**: 全选/取消全选
  - 提供"全选"按钮，一键选中所有库
  - 已全选时显示"取消全选"
- **FR-3**: 确认/取消操作
  - 标题栏右侧显示"确认"按钮（主要操作）
  - 关闭按钮(X)作为"取消"操作
  - 确认后：更新 selectedLibraryIdsProvider，切换到 latest 模式，刷新视频列表，关闭弹窗
  - 取消后：关闭弹窗，不做任何更改
- **FR-4**: 收藏夹入口
  - 收藏夹保持单选逻辑：点击即切换到收藏夹模式并关闭弹窗
  - 不参与多选

## States
- **未选中**：默认边框，浅背景
- **已选中**：主色边框，主色背景淡色填充，内部显示 ✓ 图标
- **无选中**：确认按钮禁用状态
- **全选**：全选按钮变为"取消全选"

## Acceptance Criteria

### AC-1: 多选功能正常
- **Given**: 媒体库选择弹窗已打开，显示多个媒体库
- **When**: 用户点击多个媒体库卡片
- **Then**: 每个被点击的卡片切换选中状态，弹窗保持打开，标题栏显示已选数量
- **Verification**: `human-judgment`

### AC-2: 确认应用选择
- **Given**: 用户已选择 2 个媒体库
- **When**: 用户点击"确认"按钮
- **Then**: 弹窗关闭，视频列表刷新，显示 2 个库的视频混合列表
- **Verification**: `human-judgment`

### AC-3: 取消不应用选择
- **Given**: 用户已选择 2 个媒体库
- **When**: 用户点击 X 关闭按钮
- **Then**: 弹窗关闭，选择不被应用，视频列表保持不变
- **Verification**: `human-judgment`

### AC-4: 收藏夹保持单选
- **Given**: 媒体库选择弹窗已打开
- **When**: 用户点击收藏夹卡片
- **Then**: 立即切换到收藏夹模式，弹窗关闭，不影响已选择的媒体库
- **Verification**: `human-judgment`

### AC-5: 全选功能正常
- **Given**: 媒体库选择弹窗已打开
- **When**: 用户点击"全选"按钮
- **Then**: 所有媒体库变为选中状态，按钮文字变为"取消全选"
- **Verification**: `human-judgment`
