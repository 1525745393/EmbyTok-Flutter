# 全屏按钮移动到顶部操作区 - 验证清单

## 功能验证

- [ ] Checkpoint 1: 顶部操作区显示全屏按钮
  - 视频播放页面右上角显示一个悬浮按钮
  - 图标为 `fullscreen`（竖屏模式）
  - 半透明黑色背景（`Color(0x66000000)`）
  - 圆角卡片风格（16px 圆角）

- [ ] Checkpoint 2: 点击顶部按钮切换全屏
  - 竖屏模式下点击右上角全屏按钮
  - 应用切换到横屏全屏模式，视频旋转为横屏播放
  - 图标变为 `fullscreen_exit`

- [ ] Checkpoint 3: 横屏模式下退出全屏
  - 横屏全屏模式下右上角显示退出全屏按钮
  - 图标为 `fullscreen_exit`
  - 点击后回到竖屏模式

- [ ] Checkpoint 4: 右侧操作区不显示全屏按钮
  - 右侧操作区的按钮列表中没有全屏/退出全屏按钮
  - 其他按钮顺序保持不变：连播 → 演员头像 → 点赞 → 信息 → 删除 → 倍速 → 播放模式 → 字幕 → 唱片/静音 → 下一集

- [ ] Checkpoint 5: _buildExitFullscreenButton 不再被使用
  - 代码中 `_buildExitFullscreenButton()` 方法已被删除或不再被调用
  - `grep` 搜索 `_buildExitFullscreenButton` 没有调用点

- [ ] Checkpoint 6: 连播模式下顶部操作区正常显示
  - 开启连播模式后，右侧操作区被隐藏
  - 顶部操作区的全屏按钮独立显示，不受连播模式影响
  - 点击顶部全屏按钮可切换横屏

- [ ] Checkpoint 7: 视觉风格一致性
  - 顶部操作区卡片背景与右下角连播浮层按钮使用相同的 `Color(0x66000000)` 半透明黑色背景
  - 使用相同的圆角风格
  - 图标颜色与右侧操作区按钮一致（`textPrimary`）

- [ ] Checkpoint 8: 响应式尺寸适配
  - 小屏手机：按钮尺寸适中，不显得过大
  - 大屏手机/平板：按钮尺寸不缩小，保持可点击性
  - 使用 `responsiveSize()` 方法计算尺寸

- [ ] Checkpoint 9: 动画与点击反馈
  - 顶部操作区按钮有 200ms 渐入渐出动画
  - 点击按钮时有按下缩放反馈（与 `_PressableActionButton` 效果类似）
  - 动画流畅，没有卡顿

- [ ] Checkpoint 10: 代码变更最小化
  - 新增了 `_buildTopActions()` 方法（约 20-40 行）
  - 删除了 `_buildRightActions()` 中的全屏按钮代码块（约 5 行）
  - 删除或不再调用 `_buildExitFullscreenButton()`（约 10 行）
  - 修改了 build 方法中的渲染逻辑（添加 `_buildTopActions()` 调用）
