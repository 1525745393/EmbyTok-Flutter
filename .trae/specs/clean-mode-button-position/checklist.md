# 连播模式按钮位置调整 - 验证清单

## 功能验证

- [ ] Checkpoint 1: 连播模式按钮初始位置在右下角
  - 开启连播模式后，连播 + 倍速按钮显示在视频容器右下角
  - 距离右侧边缘约 16px，距离底部约 `kBottomNavHeight + 手势条 + 16px`
  - 不遮挡视频主体内容

- [ ] Checkpoint 2: 按钮保持可拖动
  - 按下按钮后可以拖动改变位置
  - 松开后按钮停留在当前位置
  - 不能拖出屏幕/容器可视范围

- [ ] Checkpoint 3: 关闭连播模式后按钮消失
  - 点击连播开关关闭连播模式
  - 连播浮层按钮从屏幕上消失
  - 右侧操作按钮和底部信息条恢复显示

- [ ] Checkpoint 4: 横屏全屏模式下不显示连播按钮
  - 连播模式已开启后切换到横屏全屏
  - 连播浮层按钮不在全屏画面中出现

- [ ] Checkpoint 5: 按钮视觉样式
  - 按钮容器具有半透明黑色背景（如 `Color(0x66000000)`）
  - 按钮具有圆角（卡片风格）
  - 两个圆形按钮之间有合理间距（`responsiveSize(16)`）

- [ ] Checkpoint 6: 响应式尺寸适配
  - 在手机竖屏模式下：按钮尺寸适中，不显得过大
  - 在平板/桌面宽屏模式下：按钮尺寸不过小，保持可点击性
  - 按钮尺寸使用 `responsiveSize()` 方法

- [ ] Checkpoint 7: 底部安全区适配
  - 在有底部手势条的设备（iPhone 15 Pro 等）上，按钮不与手势条重叠
  - 底部边距正确包含 `kBottomNavHeight + MediaQuery.padding.bottom + 16px`

- [ ] Checkpoint 8: 动画与反馈
  - 连播模式切换时，按钮有 200-300ms 渐显/渐隐动画
  - 拖动按钮时有轻微放大效果（`_kScaleFactor = 1.1`）
  - 点击按钮时有按下缩放反馈（0.8 scale）

- [ ] Checkpoint 9: 按钮功能
  - 点击连播开关（∞ 图标）可以切换连播模式
  - 切换后显示正确的 SnackBar 提示（绿色开启 / 灰色关闭）
  - 点击倍速按钮可以循环切换倍速（0.5x → 1x → 1.25x → 1.5x → 2x）

- [ ] Checkpoint 10: 代码变更最小化
  - 变更仅出现在 `_buildCleanModeRightActions`、`_DraggableCleanActions` 及相关辅助代码
  - 非连播模式的代码逻辑不受影响
  - 右侧操作按钮、底部信息条、信息弹窗等组件未被修改

## 代码质量
