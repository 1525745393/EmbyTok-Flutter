# EmbyTok - 用户使用指南

> 本文件目标：帮助终端用户从下载 APK 到成功浏览自己的 Emby 媒体库，包含常见问题与故障排查指引。

---

## 一、下载与安装

### 1.1 获取 APK

从 GitHub Releases 页面下载最新版本：

```
https://github.com/1525745393/EmbyTok-Flutter/releases
```

根据你的手机架构选择对应的 APK：

| APK 文件名 | 适用设备 | 说明 |
|-----------|---------|------|
| `app-arm64-v8a-release.apk` | 现代 Android 手机（Pixel、三星、华为、小米、OPPO 等） | **推荐下载**，绝大多数设备用这个 |
| `app-armeabi-v7a-release.apk` | 较旧的 32 位 ARM 设备 | 兼容老手机 |
| `app-x86_64-release.apk` | Intel 64 位架构设备（少见，主要是模拟器） | 按需下载 |

如果你不确定自己手机的架构，直接下载 `app-arm64-v8a-release.apk`，若安装后打开闪退再尝试其他版本。

### 1.2 安装 APK

1. 把下载的 APK 文件复制到手机（或直接用手机浏览器下载）
2. 在手机文件管理器中点击 APK 文件
3. 首次安装时，系统会提示"未知来源应用"，按提示允许该来源安装
4. 点击"安装"，等待完成
5. 安装完成后点击"打开"，或在桌面找到 **EmbyTok** 图标启动

## 二、首次启动与配置

启动应用后，你会看到登录界面，包含四个输入项：

| 输入项 | 说明 | 示例 |
|--------|------|------|
| **后端代理地址** | FastAPI 中间层服务的地址 | `http://192.168.1.6:8000` |
| **Emby 服务器地址** | 你的 Emby 媒体服务器地址 | `http://192.168.1.6:8010` |
| **用户名** | Emby 账号的用户名 | `FK` |
| **密码** | Emby 账号的密码 | `• • • • •` |

### 2.1 关键：两个地址的区别

| 地址 | 是什么 | 典型值 |
|------|--------|--------|
| **后端代理地址** | EmbyTok 的 FastAPI 中间层（在你的电脑或服务器上运行） | `http://192.168.1.6:8000` |
| **Emby 服务器地址** | 你自己的 Emby 媒体服务（管理视频文件的那个） | `http://192.168.1.6:8010` 或 `http://192.168.1.6:8096` |

**⚠️ 常见错误**：如果把"后端代理地址"填成了 Emby 服务器地址，登录时会看到错误：

```
找不到文件 '/api/auth/login'
```

解决方法：确认后端代理服务（FastAPI）已经在电脑上启动，然后把"后端代理地址"改成它的地址。

### 2.2 启动后端代理服务

后端代理服务是 EmbyTok 的核心中间层。**在电脑上执行以下命令**：

```bash
cd EmbyTok-Flutter/backend
python3 -m venv .venv
source .venv/bin/activate              # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python3 -m uvicorn main:app --host 0.0.0.0 --port 8000
```

启动成功后，你应该看到：

```
INFO:     Uvicorn running on http://0.0.0.0:8000
INFO:     Application startup complete.
```

**重要**：`--host 0.0.0.0` 参数必须加上，否则手机无法访问到该服务。

### 2.3 验证后端是否正常

在手机浏览器访问：

```
http://192.168.1.6:8000/health
```

应该返回：

```json
{"status":"ok","version":"1.0.0","service":"embbytok-backend"}
```

### 2.4 登录

在应用的登录界面填写好四个字段后，点击红色的 **"登录"** 按钮：

- ✅ 登录成功 → 自动跳转到视频流首页
- ❌ 登录失败 → 屏幕底部显示红色错误提示，请参考本文第六节"常见问题"

## 三、基本操作：浏览视频

### 3.1 竖屏滑动（TikTok 风格）

| 操作 | 手势 |
|------|------|
| 切换下一个视频 | **向上**滑动屏幕 |
| 回到上一个视频 | **向下**滑动屏幕 |
| 显示 / 隐藏控制栏 | **单击**屏幕中央 |
| 播放 / 暂停 | **双击**屏幕中央 |
| 临时 2 倍速播放 | **长按**屏幕（释放后恢复原速度） |
| 快进 / 快退 | **水平左右拖拽**屏幕 |
| 调整亮度 | 屏幕**左侧**上下滑动 |
| 调整音量 | 屏幕**右侧**上下滑动 |

### 3.2 双视图模式

应用支持两种浏览视图，可在设置中切换：

| 视图模式 | 说明 | 适合场景 |
|---------|------|---------|
| **竖屏信息流** | 上下滑动切换视频，类似 TikTok | 快速浏览、连续刷片 |
| **媒体库网格** | 海报墙展示，点击进入播放 | 查找特定视频、浏览收藏 |

### 3.3 全屏播放

点击视频右下角的全屏按钮进入全屏模式：

- **横竖屏切换**：点击旋转图标或旋转手机自动切换
- **锁屏**：点击锁定图标防止误触
- **上一集 / 下一集**：控制栏左右箭头切换
- **清晰度切换**：设置面板中选择不同码率
- **画面比例**：自动 / 包含 / 铺满 / 16:9 / 4:3
- **倍速播放**：0.5x – 2.0x 可选
- **字幕**：支持 SRT 格式，多语言轨道切换

### 3.4 底部导航

应用底部导航栏包含以下页面：

| 图标 | 页面 | 说明 |
|------|------|------|
| 🏠 | 首页 / 视频流 | 连续播放的主入口 |
| 👤 | 演员 | 浏览演员 / 导演 / 编剧列表 |
| 🔍 | 搜索 | 按关键字搜索媒体库 |
| ❤️ | 收藏 | 浏览已收藏的视频 |
| 📜 | 观看历史 | 最近播放的视频，点击可继续播放 |
| ⚙️ | 设置 | 主题、字幕、服务器地址等 |

### 3.5 字幕

在视频播放界面：

- 点击右下角的字幕图标可以切换字幕开关
- 可选的字幕语言取决于 Emby 服务器上该视频是否有对应字幕文件（如 SRT、VTT）
- 字幕样式（字体、大小、颜色）可在"设置"中调整
- 全屏模式下可在设置面板中切换字幕轨道

### 3.6 演员 / 导演浏览

在底部导航栏点击"演员"进入：

- 三个分类 Tab：演员 / 导演 / 编剧
- 搜索框：按姓名快速查找
- 网格展示：头像 + 姓名
- 点击进入演员详情页：
  - 查看个人简介（可展开 / 折叠）
  - 浏览出演的全部作品
  - 关注 / 取消关注
  - 点击作品直接播放

### 3.7 排除已观看

设置中提供"排除已观看"选项，过滤掉已观看的视频：

- **视频流排除已观看**：媒体库设置中开启，竖屏信息流不再显示已看过的视频
- **推荐排除已观看**：推荐设置中开启，推荐列表过滤已观看内容

两个选项独立控制，可根据需要分别开启。

## 五、高级配置

### 5.1 在外面（非局域网）访问 EmbyTok

在户外使用 EmbyTok 需要让你的手机能够访问到后端代理服务。以下方案任选其一：

| 方案 | 难度 | 说明 |
|------|------|------|
| **Tailscale / ZeroTier** | ⭐ | 在电脑和手机上安装客户端，即可获得虚拟局域网地址，全程加密 |
| **frp 内网穿透** | ⭐⭐ | 在公网 VPS 上部署 frp，把本地 8000 端口转发出去 |
| **Cloudflare Tunnel** | ⭐ | 零配置暴露本地服务到公网，有域名即可 |
| **路由器端口转发** | ⭐⭐ | 在路由器上把 8000 端口映射到运行后端的电脑 |

例如使用 Tailscale 后，后端代理地址可能变成：

```
http://100.x.x.x:8000
```

### 5.2 切换 Emby 账号 / 服务器地址

在应用内的"设置"页面：

1. 点击右上角齿轮图标进入设置
2. 修改"后端代理地址"或"Emby 服务器地址"
3. 重新登录即可生效

### 5.3 主题切换

在"设置"页面可以选择：

- **跟随系统**（默认）
- **浅色主题**
- **深色主题**

推荐使用深色主题以获得更好的视频观看体验。

## 六、常见问题（FAQ）

### ❓ 问题 1：提示"找不到文件 '/api/auth/login'"

**可能原因**：你把"后端代理地址"填成了 Emby 服务器的地址。Emby 服务器没有 `/api/auth/login` 这个 API，它是 FastAPI 中间层提供的。

**排查步骤**：
1. 检查后端代理服务是否在电脑上运行
2. 在手机浏览器访问 `http://电脑IP:8000/health`，确认返回 JSON
3. 确认应用中"后端代理地址"填写的是 FastAPI 服务地址（通常是 `http://电脑IP:8000`）

**解决方案**：在登录界面把"后端代理地址"改为 `http://192.168.1.6:8000`（替换为你电脑的实际 IP）。

---

### ❓ 问题 2：提示"网络连接失败，请检查服务器地址"

**可能原因**：
- 手机和电脑不在同一局域网
- 电脑防火墙拦截了 8000 端口
- IP 地址或端口填写错误
- 后端代理服务未启动

**排查步骤**：
1. 在手机浏览器访问 `http://电脑IP:8000/health`，看能否打开
2. 确认电脑和手机连接的是同一个 WiFi 网络
3. 检查电脑防火墙是否放行 8000 端口（Linux: `ufw allow 8000`，Windows: Windows Defender 防火墙 → 允许应用通过）
4. 确认电脑上 `uvicorn main:app --host 0.0.0.0 --port 8000` 命令仍在运行

**解决方案**：按上述步骤逐一排查。最常见的是忘记加 `--host 0.0.0.0` 参数。

---

### ❓ 问题 3：视频无法播放 / 黑屏 / 转圈

**可能原因**：
- Emby 服务无法访问视频文件（路径错误 / 硬盘未挂载）
- 视频编码格式手机播放器不支持
- 网络带宽不足（大视频文件需要稳定连接）

**排查步骤**：
1. 在电脑上用浏览器直接打开 Emby 的 Web 界面，看能否播放同一个视频
2. 如果 Emby Web 也无法播放 → 是 Emby 服务的问题
3. 如果 Emby Web 可以播放但 EmbyTok 不行 → 可能是视频编码兼容性问题，尝试用 H.264 编码的视频

---

### ❓ 问题 4：字幕不显示 / 乱码

**可能原因**：
- Emby 服务器上该视频没有对应的字幕文件
- 字幕文件编码不是 UTF-8（导致中文乱码）
- 应用中字幕功能被关闭

**排查步骤**：
1. 在 Emby Web 界面播放同一视频，确认字幕可用且中文正常
2. 在 EmbyTok 中点击字幕图标切换开关
3. 若显示乱码 → 把字幕文件（如 `.srt`）转换为 UTF-8 编码后重新上传到 Emby

---

### ❓ 问题 5：视频流加载慢 / 切换卡

**可能原因**：
- WiFi 信号弱（5GHz 比 2.4GHz 快很多）
- 后端代理服务和 Emby 服务在同一台电脑上但 CPU/硬盘性能不足
- 视频文件过大（4K 蓝光原盘对网络和 CPU 要求较高）

**建议**：
1. 确保手机使用 5GHz WiFi
2. 视频文件建议预先转码为 H.264 + AAC（Emby 服务可配置硬件转码）
3. 合理使用预加载和缓存

---

### ❓ 问题 6：登录提示"用户名或密码无效"

**可能原因**：
- Emby 账号用户名或密码错误
- Emby 服务要求的是用户名（不是邮箱）

**排查步骤**：
1. 在 Emby Web 界面用同样的用户名+密码测试
2. 确认没有大小写错误
3. 确认 Emby 服务的用户管理中该账户已启用

---

### ❓ 问题 7：APK 安装后打开闪退

**可能原因**：
- 下载的 APK 架构与你的手机不匹配
- Android 版本过低（需要 Android 8.0 或更高）

**解决方案**：
1. 尝试下载其他架构的 APK（先试 `app-arm64-v8a-release.apk`，再试 `app-armeabi-v7a-release.apk`）
2. 确认你的 Android 系统版本 ≥ 8.0

---

### ❓ 问题 8：如何更新到新版本

1. 到 GitHub Releases 页面下载新 APK
2. 直接在手机上安装（新版会覆盖旧版，配置保留）
3. 启动后首次登录可能需要重新输入密码

## 七、隐私与安全

- EmbyTok 不收集任何用户数据。所有配置存储在你的手机本地。
- 用户名和密码通过 FastAPI 中间层透传给 Emby 服务，不在中间层保存。
- 登录后获得的 Access Token 也存储在手机本地。
- **建议**：在局域网内使用 HTTP 即可；若需外网访问，建议使用 HTTPS 或 Tailscale 等加密方案。
